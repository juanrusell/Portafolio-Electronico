Resumen: Autenticación basada en tokens con Java Spring Boot

La autenticación basada en tokens (como JWT) permite validar al usuario sin mantener sesiones en el servidor. Spring Security se encarga de verificar credenciales y generar un token que se devuelve al cliente. En cada solicitud posterior, el cliente incluye ese token en los encabezados, y un filtro lo valida.

Flujo básico:

— El usuario inicia sesión enviando usuario y contraseña.

— El backend verifica y genera un JWT firmado con datos del usuario.

— El cliente guarda el token y lo envía con cada solicitud.

— El backend valida el token antes de responder.

Componentes clave:

SecurityConfig: configura reglas de acceso y filtros.

UserDetailsService: carga usuarios desde la base de datos.

JwtUtil: genera y valida los tokens JWT.

JwtFilter: intercepta peticiones para verificar el token.

Buenas prácticas:

— Encriptar contraseñas con BCrypt.

— Usar expiración corta y Refresh Tokens.

— Proteger rutas con anotaciones como @PreAuthorize.

Reflexión: Comprendí cómo implementar autenticación moderna en backend con Java. Este tipo de enfoque es esencial para aplicaciones web seguras, escalables y eficientes.
