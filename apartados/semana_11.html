Temas y actividades

Temas aprendidos:

Uso de JSP (Jakarta Server Pages) como motor de vistas en aplicaciones Java, integrándolo con Spring MVC para construir interfaces web dinámicas.

¿Qué es JSP?

Es una tecnología de Java que permite generar contenido HTML de forma dinámica. Permite incluir lógica Java dentro de páginas web usando etiquetas especiales como <% %> y expresiones como ${} a través de JSTL (Jakarta Standard Tag Library).

¿Cómo se integra JSP con Spring?

Spring MVC utiliza controladores para enviar datos a las vistas JSP. Estas vistas se ubican en la carpeta /WEB-INF/views/ y el archivo de configuración indica qué motor de vista se usará (como InternalResourceViewResolver).

Flujo típico de una app con Spring + JSP:

1. Un controlador @Controller recibe la petición HTTP.

2. El controlador retorna el nombre de una vista (por ejemplo, "home").

3. Spring busca el archivo JSP correspondiente, como /WEB-INF/views/home.jsp.

4. JSP renderiza el contenido dinámico usando variables del modelo.

Ejemplo práctico realizado:

Se implementó una página de bienvenida con JSP que muestra el nombre del usuario obtenido desde el backend. Se utilizó un controlador con Model.addAttribute() para pasar el valor, y JSP lo mostró con ${nombre}.

Buenas prácticas:

— Usar JSTL en lugar de código Java embebido para mantener claridad.

— Separar lógica del backend y presentación para mantener el patrón MVC.

— Definir rutas amigables en controladores Spring.

Reflexión: Comprendí cómo usar JSP como motor de plantillas en aplicaciones Spring. Aunque actualmente hay motores más modernos como Thymeleaf, el uso de JSP sigue siendo útil para comprender el flujo entre controlador y vista en una arquitectura MVC.
