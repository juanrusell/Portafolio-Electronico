Temas y actividades

Temas aprendidos:

Creación de servicios RESTful usando Java Spring Boot, implementación de controladores, servicios, repositorios, entidades JPA y pruebas con herramientas como Postman.

¿Qué es un servicio REST?

Es un estilo arquitectónico que permite la comunicación entre sistemas mediante operaciones HTTP (GET, POST, PUT, DELETE). Spring Boot permite construir estas APIs fácilmente usando anotaciones como @RestController y @RequestMapping.

Componentes principales en un backend con Spring:

@RestController: define clases que manejarán las peticiones HTTP.

@Service: contiene la lógica de negocio que será reutilizada.

@Repository: interactúa con la base de datos usando Spring Data JPA.

@Entity: representa una tabla en la base de datos.

Ejemplo práctico realizado:

Se creó una API para gestionar productos. El modelo Producto tenía atributos como id, nombre y precio. A través del controlador, se implementaron rutas para listar, crear, actualizar y eliminar productos, usando una base de datos en memoria H2.

Pruebas y herramientas usadas:

— Postman para probar rutas HTTP.

— Swagger para documentar el API.

— Spring DevTools para recarga automática.

Reflexión: Aprendí cómo estructurar un proyecto backend profesional usando buenas prácticas como separación por capas, inyección de dependencias y anotaciones de Spring. Esto es fundamental para construir APIs robustas, mantenibles y listas para producción.
